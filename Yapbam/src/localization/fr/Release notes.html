Version 0.9.7 (??/11/2011)

Nouveautés de cette version

  • Il est désormais possible d'afficher les soldes relatifs dans l'onglet des transactions (Il suffit de cliquer sur un solde pour en faire la référence pour les autres soldes).
  • Les composants de l'onglet "opérations" sont réarrangés. Les boutons de création/édition/suppression d'opérations sont à présent sous la liste, comme dans les listes affichées dans l'onglet "Administration".
    Ajout d'un bouton pour créer les opérations en masse. Le menu de sélection des colonnes est maintenant en haut à droite, ce qui semble une position plus classique (idem dans la vue relevé de compte).
  • Correction d'un bug : La saisie d'une opération datée d'avant le 1/1/1970 provoquait des erreurs.
  • Correction d'un bug : Le filtre n'était jamais enregistré lorsque les données n'étaient pas enregistrées avant la fermeture de la fenêtre de Yapbam.
  • Correction d'un bug : L'importation d'un fichier vide provoquait une erreur.
  • Correction d'un bug : Cliquer sur un bouton d'aide provoquait un plantage si java était dans l'incapacité d'ouvrir un navigateur web.

Versions précédentes

Version 0.9.6 (23/10/2011)

  • Correction d'un bug : L'affichage de l'historique du solde d'un compte possédant un solde initial nul et uniquement des opérations de montant nul provoquait une erreur.
  • Correction d'un bug dans l'assistant de saisie des sous-opérations : Sélectionner un libellé qui n'apparraissait dans aucune opération du compte de l'opération éditée entrainait le plantage de l'assistant.

Version 0.9.5 (16/10/2011)

  • La sauvegarde du filtre entre deux lancements de Yapbam est de nouveau disponible pour les fichiers de données cryptés.
  • Correction de bug : Sous windows, lorsque les fichiers situés dans le répertoire data étaient cachés, l'état et/ou les préférences n'étaient pas sauvegardés. De même, le convertisseur de devise ne fonctionnait pas.
  • Ajout de messages d'erreur spécifiques en cas d'échec de la sauvegarde de l'état au démarrage et des préférences.

Version 0.9.4 (02/10/2011)

  • Correction d'un bug : L'export ne fonctionnait pas si on sélectionnait les commentaires dans les champs à exporter.
  • Règle un problème avec le JDK 1.7 : La fermeture de Yapbam ne fermait pas le processus de la machine virtuelle java.

Version 0.9.3 (25/09/2011)

  • Correction d'un bug dans la saisie des montants, pour les pays où le séparateur de milliers est le point (Belgique, par exemple). Lorsqu'on saisissait un nombre sans unité monétaire (exemple 5,25), ce nombre était transformé en un nombre différent à la sortie du champ (exemple 525 €).
  • Correction d'un bug dans la vue "Relevé de comptes" : Lorsqu'on ouvrait plusieurs comptes pendant l'éxécution de Yapbam, l'édition des opérations entrainait l'apparition du dialogue d'édition de l'opération plusieurs fois.

Version 0.9.2 (18/09/2011)

  • Correction d'un bug majeur de la version 0.9.0. Quand Yapbam était configuré pour mémoriser le filtre entre deux lancements, et que Yapbam était quitter alors que le fichier ouvert était protégé par un mot de passe, il était impossible de redémarrer Yapbam. La seule façon de réouvrir le fichier était de supprimer la ligne "filter" dans le fichier Data/.yapbam.
    Afin de régler le plus rapidement possible ce problème, et dans la mesure où il est causé par une erreur de conception de la sauvegarde du filtre, cette fonctionnalité est désactivé, dans cette version, pour les fichiers protégés par mot de passe. Elle sera réintroduite dans une version future.
  • Correction d'un bug dans l'importation : Lorsque le fichier importé contenait des erreurs et que la dernière colonne du fichier n'était attachée à aucune attribut d'opération, l'importation plantait.

Version 0.9.1 (16/09/2011)

  • Correction d'un bug mineur dans la saisie des modes de paiement : Le bouton "valider" n'était pas toujours correctement invalidé quand les champs n'étaient pas renseignés. La validation dans cette configuration provoquait une erreur.

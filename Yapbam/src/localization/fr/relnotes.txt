known
	Yapbam, lorsqu'il est exécuté par Java 1.6, plante quand on sélectionne un raccourci vers un dossier dans le menu déroulant d'un dialogue de sélection de fichier.<br/>En fait, ce n'est pas un bug de Yapbam, mais un <a href="http://bugs.sun.com/bugdatabase/view_bug.do;jsessionid=518b010bce71d8547e0dfeee596a?bug_id=6550546">bug de java 1.6</a>. La meilleure façon de régler le problem est d'utiliser Java 1.7.

version	0.9.2 (18/09/2011)
fix
	Correction d'un bug majeur de la version 0.9.0. Quand Yapbam était configuré pour mémoriser le filtre entre deux lancements, et que Yapbam était quitter alors que le fichier ouvert était protégé par un mot de passe, il était impossible de redémarrer Yapbam. La seule façon de réouvrir le fichier était de supprimer la ligne "filter" dans le fichier Data/.yapbam.<br/>Afin de régler le plus rapidement possible ce problème, et dans la mesure où il est causé par une erreur de conception de la sauvegarde du filtre, cette fonctionnalité est désactivé, dans cette version, pour les fichiers protégés par mot de passe. Elle sera réintroduite dans une version future.
	Correction d'un bug dans l'importation : Lorsque le fichier importé contenait des erreurs et que la dernière colonne du fichier n'était attachée à aucune attribut d'opération, l'importation plantait.

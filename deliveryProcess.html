Yapbam delivery process

  1. Set the version number in the build.xml file => version
  2. Build the classic distribution with build.xml.
  3. Upload the generated .zip file to a linux platform and change the "execute attribute" of "yapbam.sh" file.
    To do that :
    1. Extract yapbam.sh from the ubuntu archive manager.
    2. Change the attribute.
    3. Drag and drop the file to the archive in the archive manager window.
  4. Test the the archive is ok (extract its content to a new folder, and try yapbam from it).
  5. Upload the zip file to sourceforge.
  6. Test that the archive on sourceforge is ok ... sometimes, upload results in a partial file.
  7. In the sourceforge file browser, give the linux, BSD, etc... attributes (all except windows) to the new file.
  8. Create the windows installer with NSIS.
    select the "Installer based on a zip file" option.
  9. Upload the windows installer on sourceforge, test it and give it the "windows" attribute.
  10. Copy the modified archive and the updater to a folder "updateversion" in the web site.
  11. Change the version information in the class srcTests/net.yapbam.update.GetHashAndSize.
    Launch this class, then copy the output at the end of the file updateInfo.php on the web site.
    Test that the autoupdate is ok with a copy of the previous version of yapbam.
  12. Commit pending changes to svn repository.
  13. Post a comment on twitter.
  14. Post replies on opened bug report or feature requests.
  15. Update the pad_file.xml at the web site root.

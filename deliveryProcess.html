Yapbam delivery process

  1. Set the version number in the build.xml file => version
  2. Build the classic distribution with build.xml.
  3. Upload the generated .zip file to a linux platform and change the "execute attribute" of "yapbam.sh" file.
    To do that :
    1. Extract yapbam.sh from the ubuntu archive manager.
    2. Change the attribute.
    3. Drag and drop the file to the archive in the archive manager window.
  4. Test that the archive is ok (extract its content to a new folder, and try yapbam from it).
  5. Copy back the modified archive on a windows platform.
  6. Create the windows installer with NSIS.
    select the "Installer based on a zip file" option and enter "$EXEDIR\Yapbam" as default folder.
  7. Run the deployer in project YapbamDeployment (Be aware to set the right versions in the program arguments).
    Wait for it completes (it is long as this class upload Yapbam to sourceForge).
  8. Test that the autoupdate is ok with a copy of the previous version of yapbam. Also test that the archives on sourceforge are ok ... sometimes, upload results in a partial file.
  9. In the sourceforge file browser, give the linux, BSD, etc... attributes (or windows) to the new uploaded files.
  10. Commit pending changes to svn repository.
  11. Post a comment on twitter.
  12. Post replies on opened bug report or feature requests.
  13. Mise à jour du référencement par anshare (www.anshare.org)
    1. Mettre à jour la fiche logiciel :
      1. Mettre un lien vers cet .exe (les liens de téléchargement de sourceforge sont interdits) : http://www.yapbam.net/directDownload/yapbam-version.exe.
      2. Supprimer Linux dans les OS cible (l'auteur du site ne connait pas le principe de Write Once Run Everywhere, donc on n'a droit qu'à un seul OS).
